# PLANS

Use this file for multi-step execution planning standards in this repository.

## Planning Rules

- Keep milestones small enough to complete in one execution loop when possible.
- Define acceptance checks for each milestone.
- Record decisions and reversals with brief rationale.
- Track only durable outcomes; keep transient exploration in chat.

## Task Documenting Rules

- Maintain one task file per task under `docs/tasks/`.
- Keep `docs/tasks/index.md` synchronized with current task statuses.
- Bind tasks to branches when practical to reduce context ambiguity.

## Minimal Milestone Template

For each milestone, capture:

- Goal
- Scope boundary
- Files expected to change
- Validation commands
- Done criteria
