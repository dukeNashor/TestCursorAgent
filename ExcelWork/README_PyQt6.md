# 实验室生产数据统计看板（PyQt6）

## 1. 功能覆盖
应用通过外链读取生产数据 `xlsx/xlsm`，不改动原表业务数据。

已实现能力：
1. 筛选明细：按当前筛选条件列出命中记录，并展示 `Request activity`、阶段归类、新人标记、是否满足交付判定。
2. 任务排期：按实验员统计当前未完成分子数（`AG` 为空，`Unique(C|D)`）并给出“可接新任务/已饱和”；该页不受筛选时间段影响。
3. 实验员积分：按指定时间段 + 可选筛选（实验员、WBP Code、Product ID）统计实验员积分。
4. FTE 总览：仅统计老员工；该页仅展示按实验员 FTE，并额外给出“尚未获得的积分”（`AG` 为空记录的 `AI` 积分之和）。按实验员 FTE 的主口径为 `时间段内完成的FCT积分 / 时间段内完成的总积分`。
5. 人效：仅统计老员工；单一口径为 `已完成分子数 / 实验员集合FTE值之和`。其中“已完成分子”要求同时满足 `FCT + Request activity=Bulk + AG非空且在时间段内`，并按唯一 `C|D` 计数。
6. TAT 计算：统计当前所选 FCT 分子从 `Pilot起始(AF)` 到 `Bulk完成(AG)` 的天数，仅纳入在所选时间段内 Bulk 完成的分子；`TAT>10` 会标红，并给出总分子数与标红分子数。
7. 右侧统一图表区：随当前 Tab 自动切换图表；FTE 总览页显示按实验员 FTE 图，TAT 页显示 TAT 柱状图。
8. 列解析兜底：当必需字段自动匹配失败或明显读错时，自动弹出手动列匹配窗口，并在本次运行期间复用用户确认过的映射。

## 2. 字段映射
- 固定列：
  - `A`：偶联类型
  - `C`：WBP Code
  - `D`：Product ID
  - `E`：实验员
  - `AF`：起始日期
  - `AG`：完成日期（空 = 未完成）
  - `AI`：实验积分（score）
- 动态按第 2/3 行表头查找：
  - `是否新人`：值为 `是` 视为新人，其他值视为非新人
  - `Request activity`：不区分大小写匹配；`Pilot/pilot/PILOT` 识别为 `Pilot`，`Bulk/bulk/BULK` 识别为 `Bulk`
- 自动匹配失败时：
  - 弹出窗口让用户为全部必需字段手动选择列
  - 当前程序运行期间记住这份映射，并对后续加载的文件继续尝试复用
- 数据起始行：第 4 行（1~3 行为 header）

## 3. 时间段与统计口径
- 时间段：通过“起始日期 + 结束日期”指定，使用闭区间。
- 快捷时间：支持“一键套用本周/本月”，生成后可手动微调。
- 普通筛选页的统计日期固定使用完成日期 `AG`。
- 人效页固定口径：
  - 只看 `AG`
  - `AG` 必须落入所选闭区间
  - 只有 `FCT + Request activity=Bulk + AG非空` 的记录才能让分子计为已完成
  - 人效分母为当前筛选命中的老员工实验员集合的 FTE 值之和
- TAT页固定口径：
  - 只看当前所选 `FCT偶联类型`
  - 只纳入 `AG` 落在所选闭区间内的 `Bulk` 完成分子
  - `Pilot起始` 取同一分子的最早 `AF`
  - `Bulk完成` 取同一分子的最晚 `AG`
  - 忽略实验员 / WBP / Product 筛选

## 4. FCT / 非FCT 规则
- 数据加载后，界面会读取当前表里 `A` 列出现过的偶联类型枚举值。
- 用户必须手动选择一个值作为“FCT偶联类型”。
- 被选中的那个值视为 `FCT`，其余所有 `A` 列值统一视为 `非FCT`。
- 未选择时，FTE 与人效相关页只显示提示信息，不执行计算。
